# 个人网站技术架构文档

## 1. 项目概述

本项目是一个前后端分离的个人网站，前端采用 React 18 + TypeScript 构建现代化用户界面，后端采用 Python FastAPI 提供高性能异步 API 服务，数据持久化使用 MySQL，缓存与会话使用 Redis。网站包含三大核心板块：**个人介绍**（含教育经历与生活照片）、**家庭成员介绍**（含照片）、**个人博客**（仅管理员可发布，访客只读）。系统具备完善的反爬虫机制，保护个人隐私数据与核心内容不被恶意抓取。

## 4. 数据库设计

### 4.1 数据库选型说明

采用 **MySQL 8.0** 作为主数据库，核心原因：
- 成熟稳定，生产环境运维经验丰富
- 支持 JSON 类型，可灵活存储半结构化数据
- 全文检索支持（InnoDB 全文索引）
- 性能优秀，社区生态完善
- ACID 事务保证数据一致性

```sql
-- ======================== 站点配置 ========================

-- 网站配置表（键值对存储）
CREATE TABLE site_configs (
    id              INT AUTO_INCREMENT PRIMARY KEY,
    config_key      VARCHAR(100) UNIQUE NOT NULL,
    config_value    JSON NOT NULL,
    description     VARCHAR(255),

-- ======================== 反爬虫与审计 ========================

-- 访问日志表（用于反爬虫分析，可定期归档）
CREATE TABLE access_logs (
    id              BIGINT AUTO_INCREMENT PRIMARY KEY,
    ip_address      VARCHAR(45) NOT NULL COMMENT 'IPv4/IPv6',
    user_agent      VARCHAR(500),
    request_path    VARCHAR(500) NOT NULL,
    request_method  VARCHAR(10) NOT NULL,
    response_status INT,
    request_headers JSON COMMENT '请求头快照',
    fingerprint     VARCHAR(64) COMMENT '浏览器指纹哈希',
    is_suspected_bot BOOLEAN DEFAULT FALSE COMMENT '疑似爬虫标记',
    created_at      DATETIME DEFAULT CURRENT_TIMESTAMP,
    INDEX idx_ip_created (ip_address, created_at),
    INDEX idx_fingerprint (fingerprint),
    INDEX idx_created (created_at),
    INDEX idx_bot (is_suspected_bot, created_at)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_unicode_ci
PARTITION BY RANGE (YEAR(created_at)) (
    PARTITION p2024 VALUES LESS THAN (2025),
    PARTITION p2025 VALUES LESS THAN (2026),
    PARTITION p2026 VALUES LESS THAN (2027),
    PARTITION pfuture VALUES LESS THAN MAXVALUE
);

-- 爬虫封禁记录表
CREATE TABLE bot_blocks (
    id              INT AUTO_INCREMENT PRIMARY KEY,
    ip_address      VARCHAR(45) NOT NULL,
    fingerprint     VARCHAR(64),
    block_reason    VARCHAR(50) NOT NULL COMMENT 'rate_limit/captcha_fail/behavior/manual',
    evidence        JSON COMMENT '触发封禁的证据',
    expires_at      DATETIME NOT NULL COMMENT '自动解封时间',
    created_at      DATETIME DEFAULT CURRENT_TIMESTAMP,
    INDEX idx_ip (ip_address),
    INDEX idx_expires (expires_at)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_unicode_ci;
```

## 5. API 接口设计

### 5.1 接口规范

- **基础路径**: `/api/v1`
- **数据格式**: JSON
- **认证方式**: JWT (Bearer Token)，管理端与写操作接口必需
- **时间格式**: ISO 8601 (`2024-01-15T10:30:00+08:00`)
- **分页参数**: `page` (默认1), `page_size` (默认10, 最大50)
- **统一响应结构**:
  ```json
  {
    "code": 200,
    "message": "success",
    "data": {},
    "request_id": "uuid-for-tracing"
  }
  ```

### 6.1 多层防御体系

采用 **五层防御模型**，从外到内逐层过滤：

```
┌─────────────────────────────────────────────────────────────┐
│  L1: 网络层防御                                              │
│     • CDN/WAF 层拦截已知恶意 IP                               │
│     • Nginx 限制单 IP 连接数与请求频率                         │
│     • 地理区域封禁（可选）                                     │
├─────────────────────────────────────────────────────────────┤
│  L2: 请求特征防御                                            │
│     • User-Agent 黑名单校验                                   │
│     • HTTP Header 完整性检查（Referer, Accept-Language）       │
│     • TLS/JA3 指纹异常检测                                    │
├─────────────────────────────────────────────────────────────┤
│  L3: 行为分析防御                                            │
│     • 请求时间间隔分析（人类行为模式）                           │
│     • 鼠标/滚动事件采集（前端）                                │
│     • 访问路径深度与合理性检测                                │
├─────────────────────────────────────────────────────────────┤
│  L4: 验证挑战防御                                            │
│     • 图形验证码（高频请求触发）                               │
│     • 滑动验证码（可疑行为触发）                               │
│     • 数学/逻辑题验证                                        │
├─────────────────────────────────────────────────────────────┤
```

#### 6.2.2 FastAPI 反爬虫中间件

```python
# backend/app/middleware/anti_spider.py
import time
import hashlib
import asyncio
import random
from fastapi import Request, Response
from starlette.middleware.base import BaseHTTPMiddleware
from redis.asyncio import Redis

class AntiSpiderMiddleware(BaseHTTPMiddleware):
    """
    反爬虫中间件：行为分析 + 动态评分 + 分级处置
    """
    
    def __init__(self, app, redis: Redis):
        super().__init__(app)
        self.redis = redis
        
    async def dispatch(self, request: Request, call_next):
        client_ip = request.client.host
        user_agent = request.headers.get("user-agent", "")
        request_id = request.headers.get("x-request-id", "")
        
        # 1. 检查 IP 是否在封禁列表
        block_key = f"block:{client_ip}"
        if await self.redis.exists(block_key):
            return Response(
                content='{"code":403,"message":"Access denied"}',
                status_code=403,
                media_type="application/json"
            )
        
        # 2. 浏览器指纹生成（多维度特征聚合）
        fingerprint = self._generate_fingerprint(request)
        request.state.fingerprint = fingerprint
        
        # 3. 请求频率检测（令牌桶算法）
        is_limited = await self._check_rate_limit(client_ip, request.url.path)
        if is_limited:
            # 触发验证码挑战
            return Response(
                content='{"code":429,"message":"Rate limit exceeded","challenge":"captcha"}',
                status_code=429,
                media_type="application/json"
            )
        
        # 4. 行为评分（累计 suspicious 行为）
        bot_score = await self._calculate_bot_score(request, fingerprint)
        if bot_score >= 80:
            # 高风险：临时封禁 1 小时
            await self.redis.setex(block_key, 3600, "behavior_score")
            await self._log_block_event(client_ip, fingerprint, "behavior", bot_score)
            return Response(
                content='{"code":403,"message":"Suspicious activity detected"}',
                status_code=403,
                media_type="application/json"
            )
        elif bot_score >= 50:
            # 中风险：响应添加延迟（降速）
            await asyncio.sleep(random.uniform(2, 5))
        
        # 继续处理请求
        response = await call_next(request)
        
        # 5. 响应头添加追踪标记
        response.headers["X-Request-Id"] = request_id
        response.headers["X-Bot-Score"] = str(bot_score)
        
        return response
    
    def _generate_fingerprint(self, request: Request) -> str:
        """基于多维度请求特征生成指纹"""
        components = [
            request.client.host,
            request.headers.get("user-agent", ""),
            request.headers.get("accept", ""),
            request.headers.get("accept-language", ""),
            request.headers.get("accept-encoding", ""),
            request.headers.get("sec-ch-ua", ""),
        ]
        raw = "|".join(components)
        return hashlib.sha256(raw.encode()).hexdigest()[:32]
    
    async def _check_rate_limit(self, ip: str, path: str) -> bool:
        """令牌桶限流检查"""
        key = f"rate_limit:{ip}:{path.split('?')[0]}"
        
        now = time.time()
        bucket_data = await self.redis.hgetall(key)
        
        bucket = bucket_data if bucket_data else {"tokens": "10", "last_update": str(now)}
        
        # 令牌桶逻辑（简化版）
        tokens = float(bucket.get("tokens", "10"))
        last_update = float(bucket.get("last_update", str(now)))
        
        # 补充令牌：每秒恢复 2 个
        tokens = min(20, tokens + (now - last_update) * 2)
        
        if tokens < 1:
            return True  # 触发限流
        
        tokens -= 1
        await self.redis.hset(key, mapping={"tokens": str(tokens), "last_update": str(now)})
        await self.redis.expire(key, 60)
        
        return False
    
    async def _calculate_bot_score(self, request: Request, fingerprint: str) -> int:
        """计算爬虫嫌疑评分（0-100）"""
        score = 0
        pipe = self.redis.pipeline()
        
        # 维度 1: 同一 IP 短时间内请求次数
        ip_key = f"req_count:{request.client.host}"
        pipe.incr(ip_key)
        pipe.expire(ip_key, 60)
        
        # 维度 2: 指纹关联的 IP 数量（异常则加分）
        fp_key = f"fingerprint:{fingerprint}"
        pipe.sadd(fp_key, request.client.host)
        pipe.scard(fp_key)
        
        results = await pipe.execute()
        req_count = results[0]
        fp_ip_count = results[3]
        
        # 评分逻辑
        if req_count > 100:   # 1 分钟内超过 100 请求
            score += 40
        elif req_count > 50:
            score += 20
            
        if fp_ip_count > 5:   # 同一指纹关联多个 IP（代理池特征）
            score += 30
        elif fp_ip_count > 2:
            score += 10
        
        # 维度 3: Header 异常检测
        headers = request.headers
        if not headers.get("accept-language"):
            score += 10
        if not headers.get("referer") and request.url.path != "/":
            score += 5
        if "sec-fetch-site" not in headers:
            score += 10  # 可能是直接 curl
            
        return min(100, score)
    
    async def _log_block_event(self, ip: str, fingerprint: str, reason: str, score: int):
        """记录封禁事件到数据库（异步）"""
        # 使用后台任务写入数据库
        pass
```

#### 6.2.3 照片 URL 时效签名机制

```python
# backend/app/utils/image_sign.py
import time
import hmac
import hashlib
import base64
from urllib.parse import urlparse

class ImageUrlSigner:
    """照片 URL 时效签名，防止批量直链爬取"""
    
    def __init__(self, secret: str, expire_seconds: int = 600):
        self.secret = secret
        self.expire_seconds = expire_seconds
    
    def sign(self, file_path: str) -> str:
        """为图片路径生成时效签名 URL"""
        expire_ts = int(time.time()) + self.expire_seconds
        data = f"{file_path}:{expire_ts}"
        signature = hmac.new(
            self.secret.encode(),
            data.encode(),
            hashlib.sha256
        ).hexdigest()[:16]
        
        return f"{file_path}?sign={signature}&t={expire_ts}"
    
    def verify(self, file_path: str, sign: str, t: str) -> bool:
        """验证签名是否有效"""
        try:
            expire_ts = int(t)
            if time.time() > expire_ts:
                return False
            
            data = f"{file_path}:{expire_ts}"
            expected = hmac.new(
                self.secret.encode(),
                data.encode(),
                hashlib.sha256
            ).hexdigest()[:16]
            
            return hmac.compare_digest(sign, expected)
        except (ValueError, TypeError):
            return False
```

#### 6.2.4 前端反爬措施

```typescript
// 前端动态 Token 生成与验证
class ContentGuard {
    private secret: string;
    
    constructor() {
        // 从环境获取或通过接口动态下发
        this.secret = import.meta.env.VITE_API_SECRET;
    }
    
    // 生成请求签名
    generateSignature(path: string, timestamp: number): string {
        const data = `${path}:${timestamp}:${this.secret}`;
        return btoa(data);  // 生产环境使用 HMAC-SHA256
    }
    
    // 验证响应合法性
    verifyResponse(response: Response): boolean {
        const serverSign = response.headers.get('X-Server-Sign');
        const timestamp = parseInt(response.headers.get('X-Server-Time') || '0');
        
        // 检查响应时效性（防止重放缓存）
        if (Math.abs(Date.now() / 1000 - timestamp) > 30) {
            return false;
        }
        
        return true;
    }
    
    // 联系方式轻度混淆（前端渲染时解码）
    decodeContact(encoded: string): string {
        // 简单的 rot13 + base64 混淆
        return atob(encoded.split('').reverse().join(''));
    }
}

// 请求拦截器
axios.interceptors.request.use((config) => {
    const timestamp = Math.floor(Date.now() / 1000);
    const guard = new ContentGuard();
    
    config.headers['X-Request-Time'] = timestamp;
    config.headers['X-Request-Sign'] = guard.generateSignature(
        config.url || '', 
        timestamp
    );
    config.headers['X-Request-ID'] = crypto.randomUUID();
```

| 策略 | 实现方式 | 防护目标 |
|------|----------|----------|
| 动态 Token | 每个详情页请求需携带时效性签名 | 防止直接接口爬取 |

| 蜜罐数据 | API 返回不可见标记字段，被爬取则告警 | 追踪爬虫来源 |
| 反调试 | 前端检测 DevTools，触发则进入调试模式 | 防止动态分析 |

### 6.3 反爬虫处置流程

```
用户请求 → 评分检测
              │
              ├─ 评分 < 30: 正常服务
              │
              ├─ 30 ≤ 评分 < 50: 响应降速（延迟 2-5s）+ 记录日志
              │
              ├─ 50 ≤ 评分 < 80: 强制验证码挑战
              │   └─ 验证通过 → 正常服务 + 扣分
              │   └─ 验证失败 → 评分 +20，重试
              │
              └─ 评分 ≥ 80: 临时封禁 IP（1h）
                  └─ 继续触发 → 延长封禁（24h → 永久）
```

## 7. 安全设计

### 7.1 认证授权

- **JWT 双 Token 机制**: Access Token (15分钟) + Refresh Token (7天)
- **密码安全**: bcrypt 哈希，成本因子 12
- **会话管理**: Redis 存储会话状态，支持强制下线
- **接口权限**: RBAC 模型，区分 admin/editor 角色
- **博客写保护**: 所有写操作（POST/PUT/DELETE）必须携带有效 JWT

### 7.2 数据安全

- **SQL 注入防护**: SQLAlchemy ORM 参数化查询
- **XSS 防护**: 前端 React 自动转义 + 后端输入校验
- **CSRF 防护**: SameSite Cookie + Referer 校验
- **文件上传安全**: 限制文件类型（jpg/png/webp）、文件大小（最大 10MB）、存储路径隔离
- **敏感信息**: 数据库密码等使用环境变量，禁止硬编码

### 7.3 传输安全

- 强制 HTTPS（HSTS 头）
- TLS 1.2+ 版本
- 照片资源访问带时效签名

## 9. 部署架构

### 9.1 Docker Compose 编排

```yaml
# docker-compose.yml
version: "3.8"

services:
  nginx:
    image: nginx:alpine
    ports:
      - "80:80"
      - "443:443"
    volumes:
      - ./nginx/nginx.conf:/etc/nginx/nginx.conf
      - ./frontend/dist:/usr/share/nginx/html
      - ./uploads:/var/www/uploads
      - ./data/certbot:/etc/letsencrypt
    depends_on:
      - backend
    restart: unless-stopped

  backend:
    build: ./backend
    environment:
      - DATABASE_URL=mysql+aiomysql://user:pass@mysql:3306/mydb?charset=utf8mb4
      - REDIS_URL=redis://redis:6379
      - JWT_SECRET=${JWT_SECRET}
      - CAPTCHA_SECRET=${CAPTCHA_SECRET}
      - IMAGE_SIGN_SECRET=${IMAGE_SIGN_SECRET}
      - UPLOAD_DIR=/app/uploads
    volumes:
      - ./uploads:/app/uploads
    depends_on:
      - mysql
      - redis
    restart: unless-stopped
    deploy:
      resources:
        limits:
          cpus: "1.0"
          memory: 512M

  mysql:
    image: mysql:8.0
    environment:
      - MYSQL_DATABASE=mydb
      - MYSQL_USER=user
      - MYSQL_PASSWORD=${DB_PASSWORD}
      - MYSQL_ROOT_PASSWORD=${DB_ROOT_PASSWORD}
    volumes:
      - mysql_data:/var/lib/mysql
      - ./scripts/init_db.sql:/docker-entrypoint-initdb.d/init.sql
    command: --default-authentication-plugin=mysql_native_password --character-set-server=utf8mb4 --collation-server=utf8mb4_unicode_ci
    restart: unless-stopped

  redis:
    image: redis:7-alpine
    volumes:
      - redis_data:/data
    restart: unless-stopped
    command: redis-server --maxmemory 128mb --maxmemory-policy allkeys-lru

volumes:
  mysql_data:
  redis_data:
```

### 9.2 部署流程

1. **准备环境**: 安装 Docker & Docker Compose
2. **配置域名**: DNS 解析到服务器 IP
3. **申请证书**: Certbot 自动申请 Let's Encrypt SSL
4. **配置环境变量**: 复制 `.env.example` 为 `.env` 并填充
5. **初始化数据库**: `docker-compose exec mysql mysql -u user -p mydb < /docker-entrypoint-initdb.d/init.sql`
6. **构建启动**: `docker-compose up -d --build`
7. **验证服务**: 检查日志与健康检查端点

## 10. 监控与运维

### 10.1 日志采集

- **访问日志**: Nginx access.log → 按天轮转
- **应用日志**: FastAPI 结构化 JSON 日志
- **错误追踪**: Sentry 集成（可选）
- **审计日志**: 管理操作全量记录

### 10.2 告警规则

| 场景 | 阈值 | 响应 |
|------|------|------|
| 爬虫攻击激增 | 1 分钟内 429 响应 > 100 次 | 自动提升封禁等级 |
| 数据库连接耗尽 | 活跃连接 > 80% | 告警 + 自动扩容 |
| 磁盘空间不足 | 使用率 > 85% | 告警 + 清理日志 |
| 服务不可用 | 健康检查失败 | 自动重启容器 |
